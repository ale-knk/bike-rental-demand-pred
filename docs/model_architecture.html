Transformer-Based Model for Bike Demand Prediction

This page introduces a model designed to predict bike-sharing routes. It begins by defining the mathematical representations of stations and bikes, followed by modeling the routes between stations using a weighted directed graph. Next, the problem of predicting the next route is formulated, and the model's architecture is described, incorporating embeddings, fully connected layers, and a Transformer encoder. Finally, the loss function used to train the model is detailed, and the complete architecture is summarized.

1. Representation of Stations and Bikes

  • Stations: S={si}i=1,...,nsS = \{s_i\}_{i=1,...,n_s}S={si​}i=1,...,ns​​, where each station sis_isi​ is represented by its longitude and latitude coordinates (xi,yi)(x^i, y^i)(xi,yi).
  • Bikes: B={bi}i=1,...,nbB = \{b_i\}_{i=1,...,n_b}B={bi​}i=1,...,nb​​, where each bike bib_ibi​ can be assigned to a specific station at any given time.

  • Weighted Directed Graph: A weighted directed graph G=(V,E)G = (V, E)G=(V,E) is used to model the routes between stations.

    • Vertices (V): Set of stations SSS.

    • Edges (E): Each edge eij∈Ee_{ij} \in Eeij​∈E represents a route from station sis_isi​ to station sjs_jsj​. Formally, each edge eije_{ij}eij​ is denoted as:
      eij=(si,sj,tij,tij_departure,tij_arrival)e_{ij} = (s_i, s_j, t_{ij}, t_{ij\_departure}, t_{ij\_arrival})eij​=(si​,sj​,tij​,tij_departure​,tij_arrival​)
      Where:

      • sis_isi​ and sjs_jsj​: Starting and destination stations.
      • tijt_{ij}tij​: Travel time.
      • tij_departuret_{ij\_departure}tij_departure​: Departure time.
      • tij_arrivalt_{ij\_arrival}tij_arrival​: Arrival time.

    • Weights (W): The edge weights are defined as wij=tij_arrival−tij_departurew_{ij} = t_{ij\_arrival} - t_{ij\_departure}wij​=tij_arrival​−tij_departure​.

3. Problem Definition for Prediction

The goal is to predict the next route eije_{ij}eij​ in a sequence of routes, given the previous routes. Formally, given a sequence of routes τ={ei1j1,ei2j2,...,einjn}\tau = \{e_{i_1j_1}, e_{i_2j_2}, ..., e_{i_nj_n}\}τ={ei1​j1​​,ei2​j2​​,...,ein​jn​​}, the objective is to predict the next route ein+1jn+1e_{i_{n+1}j_{n+1}}ein+1​jn+1​​.

To predict the next route ein+1jn+1e_{i_{n+1}j_{n+1}}ein+1​jn+1​​, a supervised learning model is used. The target variable is the route ein+1jn+1e_{i_{n+1}j_{n+1}}ein+1​jn+1​​, and the model features are based on the sequence of routes τ\tauτ up to time nnn.

The route eije_{ij}eij​ is denoted as a random variable EEE. Predicting the next route can be formulated as:

P(En+1=ein+1jn+1∣τ)=P(En+1=ein+1jn+1∣ei1j1,ei2j2,...,einjn)P(E_{n+1} = e_{i_{n+1}j_{n+1}} \mid \tau) = P(E_{n+1} = e_{i_{n+1}j_{n+1}} \mid e_{i_1j_1}, e_{i_2j_2}, ..., e_{i_nj_n})P(En+1​=ein+1​jn+1​​∣τ)=P(En+1​=ein+1​jn+1​​∣ei1​j1​​,ei2​j2​​,...,ein​jn​​)

Since ein+1jn+1=(sin+1,sjn+1,tin+1,tjn+1)e_{i_{n+1}j_{n+1}} = (s_{i_{n+1}}, s_{j_{n+1}}, t_{i_{n+1}}, t_{j_{n+1}})ein+1​jn+1​​=(sin+1​​,sjn+1​​,tin+1​​,tjn+1​​), we can express EEE as Sdeparture×Sarrival×Tdeparture×TarrivalS_{departure} \times S_{arrival} \times T_{departure} \times T_{arrival}Sdeparture​×Sarrival​×Tdeparture​×Tarrival​, decomposing the prediction of the next route into four probability distributions:

  • P(Sdeparture=sin+1∣τn)P(S_{departure} = s_{i_{n+1}} \mid \tau_{n})P(Sdeparture​=sin+1​​∣τn​): Prediction of the departure station sin+1s_{i_{n+1}}sin+1​​ for the next route.
  • P(Sarrival=sjn+1∣sin+1,τn)P(S_{arrival} = s_{j_{n+1}} \mid s_{i_{n+1}}, \tau_{n})P(Sarrival​=sjn+1​​∣sin+1​​,τn​): Prediction of the arrival station sjn+1s_{j_{n+1}}sjn+1​​ for the next route.
  • P(Tdeparture=tin+1jn+1_departure∣sin+1,sjn+1,τn)P(T_{departure} = t_{i_{n+1}j_{n+1}\_departure} \mid s_{i_{n+1}}, s_{j_{n+1}}, \tau_{n})P(Tdeparture​=tin+1​jn+1​_departure​∣sin+1​​,sjn+1​​,τn​): Prediction of the departure time tin+1jn+1_departuret_{i_{n+1}j_{n+1}\_departure}tin+1​jn+1​_departure​ for the next route.

Notably, the predictions are sequentially added as variables influencing the conditional distribution of the subsequent probability distribution. Thus, the predicted arrival station, in addition to depending on the sequence of routes, also depends on the predicted departure station.

4. Model Architecture

The model input is a sequence of routes τ={ei1j1,ei2j2,...,einjn}\tau = \{e_{i_1j_1}, e_{i_2j_2}, ..., e_{i_nj_n}\}τ={ei1​j1​​,ei2​j2​​,...,ein​jn​​}, where each route eije_{ij}eij​ is represented as a tuple (si,sj,tij_departure,tij_arrival)(s_i, s_j, t_{ij\_departure}, t_{ij\_arrival})(si​,sj​,tij_departure​,tij_arrival​).

5. Loss Function

To train the model, we use a combination of loss functions:

  • Categorical Cross-Entropy (CE) for departure and arrival stations.
  • Mean Squared Error (MSE) for departure and arrival times.

The total loss function is defined as:
L=LCE(psin+1,sin+1)+LCE(psjn+1,sjn+1)+LMSE(ptin+1jn+1_salida,tin+1jn+1_salida)+LMSE(ptin+1jn+1_llegada,tin+1jn+1_llegada)\mathcal{L} = \mathcal{L}_{\text{CE}}(\mathbf{p}_{s_{i_{n+1}}}, s_{i_{n+1}}) + \mathcal{L}_{\text{CE}}(\mathbf{p}_{s_{j_{n+1}}}, s_{j_{n+1}}) + \mathcal{L}_{\text{MSE}}(\mathbf{p}_{t_{i_{n+1}j_{n+1}\_salida}}, t_{i_{n+1}j_{n+1}\_salida}) + \mathcal{L}_{\text{MSE}}(\mathbf{p}_{t_{i_{n+1}j_{n+1}\_llegada}}, t_{i_{n+1}j_{n+1}\_llegada})L=LCE​(psin+1​​​,sin+1​​)+LCE​(psjn+1​​​,sjn+1​​)+LMSE​(ptin+1​jn+1​_salida​​,tin+1​jn+1​_salida​)+LMSE​(ptin+1​jn+1​_llegada​​,tin+1​jn+1​_llegada​)
